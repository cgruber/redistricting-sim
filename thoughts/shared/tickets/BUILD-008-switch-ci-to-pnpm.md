## Summary

CI currently runs `npm ci`, which requires `package-lock.json` to be committed.
Bazel's `npm_translate_lock` uses `pnpm-lock.yaml` as the authoritative dep graph.
Both lockfiles must be updated in sync on every dep change — a process smell that
was exposed when adding `react` as a devDependency in PR #153 (CI failed until
`package-lock.json` was regenerated separately). Switching CI to
`pnpm install --frozen-lockfile` removes the need for `package-lock.json` entirely.

## Current State

- `pnpm-lock.yaml` — Bazel-authoritative; generated by `npx pnpm`
- `package-lock.json` — CI-only; generated by `npm install`; must stay in sync with `package.json`
- Every dep change requires running both `npx pnpm add` and `npm install` locally

## Goals / Acceptance Criteria

- [ ] CI Buildkite step updated: replace `npm ci` with `pnpm install --frozen-lockfile`
  (or equivalent pnpm frozen-install invocation)
- [ ] `package-lock.json` removed from the repo and added to `.gitignore`
- [ ] `game/setup.sh` (local developer setup) updated if it references `npm ci`
- [ ] CI passes end-to-end after change
- [ ] `package.json` `pnpm` section confirmed correct (`onlyBuiltDependencies: []`)
